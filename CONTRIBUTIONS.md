# Contributing to the Agentic Trust Framework

ATF is an open governance specification published under CC BY 4.0. We welcome contributions that strengthen the specification and expand its utility.

## How to Contribute

### Specification Feedback
- Open issues for specification clarifications, gaps, or improvements
- Propose extensions to the five core elements or maturity model
- Suggest terminology refinements

### Implementation Experience
- Share your experience implementing ATF in production environments
- Submit implementation reports as pull requests to document what worked, what didn't, and what you learned
- Reference implementations help the community and inform specification evolution

### Crosswalk Contributions
- Propose mappings between ATF and additional compliance or governance frameworks
- Submit corrections or refinements to existing crosswalks in CROSSWALKS.md
- Include regulatory context and applicability notes

### Community Issues

## What This Repository Contains

ATF is a **specification and governance framework**, not a code library. This repository contains:
- The canonical specification documents (SPECIFICATION.md, MATURITY_MODEL.md)
- Conformance tiers and requirements (CONFORMANCE.md)
- Implementation patterns and guidance (IMPLEMENTATION_PATTERNS.md)
- Framework crosswalks (CROSSWALKS.md)
- Security principles (SECURITY.md)

## Contribution Guidelines

- Specification language should use RFC 2119 keywords (MUST, SHOULD, MAY) where appropriate
- Keep content technology-agnostic and architecture-level; vendor and library mentions belong in implementation guides, not the core specification
- Include compliance caveats where mappings are illustrative rather than prescriptive
- Reference primary sources for any framework or standard cited
